# 节点可用性检测 — 需求

## 背景
管理员需要快速了解节点是否可达，判断节点服务是否正常运行。

## 功能需求
1. 单个检测：点击节点行的检测按钮，检测该节点是否可达
2. 全部检测：点击"全部检测"按钮，串行检测当前页所有节点
3. 显示"可用"或"不可用"状态，检测中显示 loading 动画

## 技术约束
- 浏览器无法发 ICMP ping，使用后端 TCP Socket 探测
- 探测目标：节点的 address:port
- 超时时间：5 秒
- 返回结果：{ reachable: boolean, latency: number }（不可达时 latency 为 -1）
